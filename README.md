# Autonomous Deep Research AI Agent 
**全自動化深度研究 AI 代理人**

---

## 💡 專案目標與背景 (Project Objectives & Background)

**[中文]**
本專案旨在打造一款企業級的「全自動深度研究代理人 (Autonomous Research Agent)」。使用者只需輸入一個研究主題（如：產業趨勢、技術分析），AI 即會自動拆解子關鍵字、跨網搜尋、爬取並過濾相關網頁內容，最後綜合數萬字生資料，統整出一份具備高度專業性且帶有嚴格來源標註的分析報告。
專案的核心目標是解決大型語言模型 (LLM) 在實務應用上常見的兩大痛點：**「無依據的幻覺 (Hallucinations)」** 與 **「API 呼叫配額限制 (Rate Limits/Quotas)」**，並提供客戶端零門檻的 SaaS 網頁體驗。

**[English]**
This project aims to build an enterprise-grade "Autonomous Research Agent". By simply entering a research topic (e.g., industry trends, technical analysis), the AI automatically breaks it down into sub-queries, executes cross-web searches, scrapes/filters relevant web content, and synthesizes tens of thousands of words of raw data into a highly professional analytical report with strict source citations.
The core objective is to resolve two major pain points typical in Large Language Model (LLM) applications: **"Unsubstantiated Hallucinations"** and **"API Rate Limits / Quotas"**, while delivering a zero-barrier, seamless SaaS web experience for end-clients.

## 🚀 核心挑戰與解決方案 (Core Challenges & Solutions)

### 1. 模型降級與高可用性系統 (Model Fallback & High Availability System)
* **[中文]** 開發「瀑布流降級機制 (Waterfall Fallback)」。當執行過程中 Gemini 1.5 Pro 等高階模型遇到配額耗盡或 429 錯誤時，系統能即時、動態地抓取 Google 最新可用模型名單，自動判斷剩餘配額，並無縫切換至 Flash 系列或其他備援模型，搭配智慧解析 API 的等待時間進行指數退避 (Exponential Backoff)，實現完全不中斷的穩定運行。
* **[English]** Developed a "Waterfall Fallback" mechanism. During execution, if advanced models like Gemini 1.5 Pro encounter quota exhaustion or 429 Errors, the system dynamically fetches the latest available Google models, evaluates remaining quotas, and seamlessly switches to alternatives (e.g., Flash models). Paired with intelligent parsing of API wait times for exponential backoff, this ensures uninterrupted, stable operations.

### 2. 精準爬蟲與零幻覺雜訊過濾 (Precision Scraping & Zero-Hallucination Noise Reduction)
* **[中文]** 串接 Serper.dev 企業級搜尋引擎，搭配 LLM 負責意圖感知。導入了「AI Snippet 去雜訊預審機制」，在實際進入網頁抓取全文前，先由 AI 為搜尋回傳的標題與摘要（Snippet）進行 0-100 分的相關性評估。若偵測到行事曆、假期表、或無意義導購等雜訊網頁，將直接捨棄不抓全文以省算力。若結果全數不及格，更會觸發防呆機制，要求 AI 重新擬定關鍵字。撰寫報告時強制啟動「Source-Only」模式，要求論點必須附帶檔案來源。
* **[English]** Integrated the Serper.dev enterprise search engine paired with LLM intent-awareness. Introduced an "AI Snippet Pre-filtering mechanism" that scores titles and snippets for relevance (0-100) before conducting computationally expensive full-text extraction. Irrelevant noise like calendars, holiday schedules, or generic wikis are preemptively discarded. A fail-safe triggers query rewriting if all iterations score poorly. For report generation, a strict "Source-Only" mode forces the AI to append citations to every analytical point.

### 3. 多租戶雲端部署與沙盒化架構 (Multi-Tenant Deployment & Sandbox Architecture)
* **[中文]** 將純後端腳本升級為使用 Streamlit 打造的商用 Web UI。為解決多位客戶同時操作可能產生的資料覆蓋與硬碟容量問題，導入了 `tempfile.TemporaryDirectory` 的「閱後即焚沙盒機制」。所有爬蟲生資料與階段產出均存在獨立記憶體沙盒中，待使用者下載報告後即刻銷毀。此外，實作 `io.StringIO` 日誌遮罩技術，徹底解決舊版打包依賴在跨平台部署時引發的 `cp950` Console 編碼崩潰。
* **[English]** Upgraded the pure backend scripts into a commercial-grade Web UI using Streamlit. To prevent data overwriting and storage exhaustion during concurrent multi-tenant usage, a "burn-after-reading" sandbox architecture was implemented using `tempfile.TemporaryDirectory`. All raw scraped data and outputs are isolated dynamically and destroyed immediately after report download. Furthermore, `io.StringIO` stdout-masking was implemented to completely resolve `cp950` console encoding crashes caused by legacy dependencies during cross-platform deployment.

### 4. 自動化企業報告匯出 (Automated Enterprise Report Exporting)
* **[中文]** 為滿足商業客戶對於文件二次編輯的需求，拋棄了容易產生環境相容性問題的開源轉換庫，改為自行開發客製化的 Python 轉換元件 (`export.py`)。它能精準解析由 Gemini 產出的純文字 Markdown 結構（涵蓋標題、清單、粗體字等），原生匯出符合業界標準的 Word (`.docx`) 報告檔案，並於前端提供平行的雙格式一鍵下載服務。
* **[English]** To meet the secondary editing needs of commercial B2B clients, unreliable open-source conversion wrappers were discarded in favor of a custom-built Python component (`export.py`). It accurately parses plain-text Markdown tags generated by Gemini (including multi-level headings, lists, and bold text) and outputs them natively into industry-standard Word (`.docx`) file formats, offering users parallel, one-click dual-format downloads on the frontend.

## 🛠 技術堆疊 (Tech Stack)

* **Programming Language**: Python 3.10+
* **LLM & Agent Framework**: Google Gemini (Pro/Flash) / `google-generativeai`
* **Search Engine & Data Acquisition**: Serper.dev (Google Search API), `newspaper3k`, `requests`
* **Web Scraping Protection Bypass**: User-Agent spoofing, Exception routing
* **Web Framework & Cloud Hosting**: Streamlit, Streamlit Community Cloud
* **Document Parsing & Generation**: Regular Expressions (Regex), `python-docx`
* **System Engineering**: Contextlib (Context managers), Thread-safe Sandboxing (`tempfile`)
